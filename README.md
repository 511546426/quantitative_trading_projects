# A股量化交易系统 · 设计总览

> 目标：构建稳定年化 15%~30%、可控回撤、可长期复用的 A 股量化系统；
> 技术栈以 C++ 为核心，达到私募级技术含金量。

---

## 项目定位

| 维度 | 说明 |
|------|------|
| 市场 | A 股（沪深两市，未来可扩展期货/可转债） |
| 资金规模 | 个人实盘起步，策略验证后逐步加仓 |
| 收益目标 | 年化 15%~30%，最大回撤 ≤ 20% |
| 持仓风格 | 日频/周频为主，兼顾日内超短 |
| 技术定位 | C++ 执行引擎 + Python 策略研究，打造私募级架构 |

## 技术栈速览

### 研究端（Python）
- **数据获取**：Tushare Pro、AKShare、BaoStock
- **数据存储**：ClickHouse（K线）、PostgreSQL（基本面）、Redis（实时）
- **回测框架**：自研 vectorized backtester（速度优先）+ Backtrader（验证）
- **因子研究**：Alphalens、自研因子评估框架
- **参数优化**：Optuna（贝叶斯优化）+ Walk-Forward Optimization

### 执行端（C++）
- **语言标准**：C++17/20
- **网络/IO**：Asio（异步 IO）
- **行情接入**：CTP / XTP / 券商量化 API（miniQMT）
- **消息队列**：SPSC lockfree queue（自研）
- **序列化**：FlatBuffers / Protobuf
- **构建系统**：CMake + Conan
- **测试**：Google Test + Google Benchmark

---

## 核心设计原则

1. **研究与生产隔离**：Python 负责 Alpha 挖掘，C++ 负责执行，通过 ZMQ/共享内存通信
2. **回测与实盘一致**：同一份策略逻辑同时驱动回测和实盘，避免"回测幽灵"
3. **风控优先**：任何订单发出前必须过三重风控（信号层/OMS 层/券商层）
4. **可观测性**：所有关键路径打点，实时监控延迟、成交率、PnL
5. **渐进式演化**：先跑通小闭环，再逐步替换高性能组件

## 快速开始

```bash
# 克隆并初始化
git clone <repo>
cd quantitative_trading_projects

# 研究环境
python -m venv .venv && source .venv/bin/activate
pip install -r requirements-research.txt

# C++ 引擎构建
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
make -j$(nproc)
```
