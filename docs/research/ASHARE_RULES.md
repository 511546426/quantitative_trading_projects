# A 股交易规则与量化陷阱手册

> 这是 A 股量化的"避坑宝典"。很多回测看起来很美，实盘一塌糊涂，
> 根本原因是没有正确处理这些 A 股特有规则。

---

## 一、基本交易规则

### T+1 制度
```
股票：当日买入，次日才能卖出
  ├── 影响：无法日内高抛低吸（不像美股、期货）
  ├── 影响：买入后被套，当日无法止损
  └── 量化应对：回测中买入信号必须是 T+1 执行卖出逻辑

ETF：T+0 申赎，T+1 卖出（二级市场 T+1）
期货：T+0，可日内开平
```

### 涨跌停板制度
```
普通股：单日 ±10%
科创板/创业板注册制：单日 ±20%
新股（上市 5 日内）：无涨跌停限制（但有额外规则）
ST 股：单日 ±5%

量化影响：
  1. 涨停股次日可能高开，也可能低开 → 涨停不追（策略规则）
  2. 跌停股卖不出去 → 持仓浮亏可能无法止损
  3. 集合竞价无限制，但偏离中间价 ±10% 会撤回
```

### 交易时间
```
集合竞价（开盘前）：9:15~9:25
  └── 9:25 确定开盘价

连续竞价：9:30~11:30, 13:00~14:57
尾盘集合竞价：14:57~15:00
  └── 14:57 后不接受新订单（实际是 14:57 提交，15:00 成交）

量化建议：
  ├── 日频策略：收盘后计算信号 → 次日 9:30 开盘买入 或 14:57 尾盘买入
  └── 尾盘委托价格稳定，推荐用尾盘集合竞价
```

## 二、成本结构

### 完整交易成本（单边）

| 费用项 | 买入 | 卖出 | 说明 |
|--------|------|------|------|
| 佣金 | 0.02%~0.03% | 0.02%~0.03% | 可谈，大资金可低至 0.015% |
| 印花税 | 0 | **0.10%** | 仅卖出收，固定，无法谈 |
| 过户费 | 0.00002 | 0.00002 | 极小，可忽略 |
| 滑点（估计）| 0.03%~0.10% | 0.03%~0.10% | 取决于流动性 |

**单边合计**：买入 ~0.05%，卖出 ~0.15%，**一买一卖合计 ~0.20%**

### 对策略的影响

```python
# 换手率 100%/年 = 每年全部换手一次
# 成本 = 100% * 0.20% = 0.20%/年 → 几乎可以忽略

# 换手率 1200%/年 = 每月全部换手一次（月频策略常见）
# 成本 = 1200% * 0.20% = 2.4%/年 → 需要考虑

# 换手率 6000%/年 = 每周全部换手一次（周频策略）
# 成本 = 6000% * 0.20% = 12%/年 → 严重影响收益！

# 结论：高频换手策略在 A 股必须有足够高的毛收益才能覆盖成本
```

## 三、流动性陷阱

### 小市值股票的流动性问题

```python
# 回测中"买了"，实盘根本买不到或推高价格
# 过滤标准（保守）
MIN_DAILY_AMOUNT = 5_000_000   # 日成交额至少 500 万元
MIN_MV = 2_000_000_000         # 总市值至少 20 亿元（避免小盘股）

# 更保守的标准：只选沪深 300 / 中证 500 成分股
# 代价：Alpha 更少，但执行更可靠
```

### 冲击成本估算

```python
def estimate_impact_cost(order_amount: float, 
                         daily_amount: float) -> float:
    """
    order_amount: 订单金额（元）
    daily_amount: 该股票日均成交额（元）
    返回：预估冲击成本（bps）
    """
    participation_rate = order_amount / daily_amount
    # 经验公式：参与率 1% → 冲击成本约 5bps
    return participation_rate * 500  # 单位 bps

# 例：买入 100 万，日成交额 1 亿
# participation_rate = 1%
# impact_cost = 5bps = 0.05%
# 超过日成交 5% 则冲击成本显著，应当拆单或放弃
```

## 四、必须过滤的特殊股票

### ST/\*ST 股票
```python
# ST（特别处理）：涨跌停 ±5%，财务有问题
# *ST：暂停上市警告
# 过滤原因：流动性差，退市风险高，散户炒作无规律

def filter_st(df: pd.DataFrame) -> pd.DataFrame:
    return df[~df['name'].str.contains('ST')]
```

### 新股/次新股
```python
# 上市 60 个交易日内不纳入选股池
# 原因：定价不稳定，新股效应扭曲，历史数据不足

MIN_LISTING_DAYS = 60
def filter_new_stocks(df: pd.DataFrame, 
                      trade_date: str) -> pd.DataFrame:
    df['listing_days'] = (pd.to_datetime(trade_date) - 
                          pd.to_datetime(df['list_date'])).dt.days
    return df[df['listing_days'] >= MIN_LISTING_DAYS]
```

### 退市/将退市股票
```python
# 过滤已公告退市整理期的股票
# 保留退市历史数据（用于回测历史），但实盘不持有
```

### 停牌股票
```python
# 当日停牌的股票不可交易
# 长期停牌（>20 日）的股票复牌后风险极大
def filter_suspended(df: pd.DataFrame) -> pd.DataFrame:
    return df[df['volume'] > 0]
```

## 五、财报时间的处理

### 财报披露时间规律

```
季报（Q1）：4 月 30 日前
半年报（Q2）：8 月 31 日前
三季报（Q3）：10 月 31 日前
年报（Q4）：次年 4 月 30 日前

注意：A 股允许延迟披露，部分公司在截止日前最后一天披露
```

### 财务数据的时效性陷阱

```python
# 错误：2023-05-01 用了 2023 年 Q1 财报数据
# 可能正确（如果 Q1 财报已发布）

# 正确做法：用 announcement_date（实际披露日期），而非 end_date（报告期末）

# Tushare 的 fina_indicator 接口提供 ann_date（公告日期）
# 应使用 ann_date 来确定哪天可以使用该数据

                         fields='ts_code,ann_date,end_date,roe')
```

## 六、指数成分股变更

```
沪深 300 / 中证 500 成分股每年调整两次（6月、12月）
调整公告前后存在"指数效应"套利机会

注意：
1. 纳入指数的股票在公告后通常上涨（指数基金被动买入）
2. 剔除指数的股票通常下跌
3. 回测时应使用历史成分股，而非现在的成分股（否则严重的幸存者偏差）
```

## 七、量化策略在 A 股的特殊考量

### A 股市场特征

```
1. 散户主导（约 60%~70% 交易量）→ 情绪因子更有效
2. 机构存在羊群效应 → 趋势动量因子有效
3. 信息不透明 → 财报超预期效应更强
4. 监管变化频繁 → 策略需有适应性
5. 政策市 → 宏观政策时间节点前后风险较大
```

### 容易失效的策略

```
1. 纯技术面高频策略：成本太高
2. 做空策略：A 股无法直接做空个股（只能做空股指期货）
3. 高换手动量策略：印花税杀伤力大
4. 纯基本面长持：A 股估值锚定性弱，价值陷阱多
```

### 相对有效的策略

```
1. 截面动量（中期）：3~12 月回报
2. 低波动因子：A 股有效（与其他市场相反方向也有效）
3. 质量因子（ROE/盈利稳定性）：长期有效
4. 规模因子（小盘股溢价）：有效但流动性差，实盘难执行
5. 大盘择时（趋势跟踪）：市场状态有明显的牛熊特征
```

## 八、实盘运营注意事项

### 每日操作时间表

```
8:30  - 检查系统状态，确认数据更新完成
9:00  - 计算当日信号（基于昨日收盘后数据）
9:15  - 集合竞价观察（不操作，仅监控）
9:30  - 开盘买入（如有信号）
      - 检查昨日持仓是否有异常（停牌/涨跌停）
11:30 - 上午收盘，检查成交情况，处理未成交订单
13:30 - 下午开盘，补单或取消过期订单
14:50 - 准备收盘委托
14:57 - 尾盘集合竞价委托（如有需要）
15:05 - 收盘后数据下载
16:00 - 每日报告（收益/持仓/信号）
```

### 紧急处理预案

```
情况 1：系统崩溃（不能自动发单）
  → 手动查看持仓，按信号手动操作
  → 重要：持仓和信号文件备份到云端

情况 2：行情异常（断网/数据源失联）
  → 不发新订单，只处理现有持仓的止损
  → 切换备用行情源（AKShare 作为备用）

情况 3：组合快速亏损
  → 检查是否有异常订单（重复下单/错误方向）
  → 触发熔断：暂停所有自动下单，手动接管
  → 记录日志，事后分析原因
```
