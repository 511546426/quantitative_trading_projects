# Sharpe > 1.5 实现方案

## 回测结论总结

| 方案 | Sharpe | 年化收益 | 波动率 | MaxDD | 可行性 |
|------|--------|--------|--------|-------|--------|
| 纯多头 (无对冲) | 0.51 | 14.8% | 29% | -44% | ✓ 可实盘，但夏普低 |
| **A. 多头 + 期货 beta 对冲** | 0.51 | 8.1% | 16% | -22% | ✓ 降回撤，但夏普不变 |
| **B. 多空个股 (融券)** | **1.62** | **48%** | 30% | -26% | ⚠ 融券限制多 |
| C. 多头 + ETF 空头 | 0.39 | 6.1% | 16% | -23% | ✓ 但额外成本拖累 |

**核心结论**：
- 期货 beta 对冲**只能降回撤，无法提升 Sharpe**（因为 alpha 本身不变）
- 达到 Sharpe 1.5 的唯一路径：**做空超买个股**，捕获"空头 alpha"
- 方案 B（融券多空）理论上可达 Sharpe **1.62**

---

## 为什么期货对冲无法提升 Sharpe？

```
Sharpe = Alpha / Residual_Vol

期货对冲: 去掉 Beta × Market_Vol
→ Alpha 不变（仍是选股收益）
→ Residual_Vol 减小（去掉系统性波动）
→ Sharpe 提升有限（Beta × Market_Return 也被去掉了）

多空个股: 做多低偏离 + 做空高偏离
→ Alpha 翻倍（空头贡献负 alpha ≈ 多头 alpha）
→ Beta 天然对消（多空方向相反）
→ Sharpe 大幅提升
```

## 方案 B：多空个股融券策略（推荐）

### 策略逻辑

```
每 21 个交易日（月频）调仓:

做多腿: 买入 MA60 偏离率最低的 100 只 (最超卖)
         → 等权 1%/只，总计 100% 多头暴露

做空腿: 融券做空 MA60 偏离率最高的 100 只 (最超买)
         → 等权 1%/只，总计 100% 空头暴露

净暴露 = 0%（市场中性）
```

### A 股融券实操约束

| 约束 | 说明 | 解决方案 |
|------|------|--------|
| 可融券标的有限 | 约 3500 只，以沪深300/500成分为主 | 从可融券列表中选空头标的 |
| 融券费率 | 年化 8-15%，高于期货成本 | 降低空头换手率 → 季频调仓空头 |
| 融券余量不足 | 大量做空同一标的会被限制 | 分散到 100 只，单只不超过 1% |
| 强制平仓风险 | 券商可随时召回融出的券 | 设置 stop-loss，备用替换池 |
| 两融账户门槛 | 需 50 万以上 + 信用账户 | 初期用期货部分对冲过渡 |

### 实现架构

```
策略信号层 (Python)
│
├── 多头信号: MA60偏离率 bottom 100
│   + RSI(14) 超卖过滤
│   + 成交额 > 1亿/日
│
├── 空头信号: MA60偏离率 top 100
│   + 仅限可融券列表
│   + 排除涨停/停牌
│
└── 仓位计算: 多空各 100%，净暴露 < 20%

风控层
│
├── 单只股票: 多头 ≤ 2%，空头 ≤ 1.5%（空头更严）
├── 单行业: 多头 ≤ 25%，空头 ≤ 20%
├── 组合回撤: > 10% 降低空头杠杆
└── 融券余量监控: < 50% 时触发替换

执行层
│
├── 多头: miniQMT 市价单，开盘集合竞价
├── 空头: 融券接口（XTP/QMT 均支持）
└── 调仓: 先卖出/平空头，再建立新头寸
```

## 过渡方案：分阶段实现

### 阶段 1：纯多头 + 期货对冲（立即可做）

**目标 Sharpe**: 0.5 → 0.7+（通过提升 alpha 质量）

```python
# 期货仓位计算
def calc_futures_hedge(portfolio_value, portfolio_beta, index_level):
    """
    portfolio_value: 股票组合总市值
    portfolio_beta:  组合 beta（约 1.05~1.10）
    index_level:     沪深300 当前点位
    
    IF合约乘数 = 300元/点
    每手合约价值 = 300 × index_level
    """
    contract_value = 300 * index_level  # e.g., 300 × 4000 = 120万
    target_short = portfolio_value * portfolio_beta
    n_contracts = int(target_short / contract_value)
    return n_contracts  # 做空 n 手 IF 合约

# 示例: 100万股票, beta=1.05, 指数4000点
# → 做空 int(100万 × 1.05 / 120万) = 0 手（资金不足一手）
# → 需要 120万 / 1.05 ≈ 114万起步才能满对冲
```

**注意**: IF 合约最小 1 手 ≈ 120 万，起步资金门槛高。  
**替代**: 用 **300ETF 融券** 作过渡（门槛更低）。

### 阶段 2：引入融券做空（3-6 个月后）

**目标 Sharpe**: 1.0 → 1.5+

```python
# 可融券池获取（miniQMT接口）
def get_shortable_stocks(trader) -> set:
    """获取当前可融券的标的列表"""
    securities = trader.query_margin_detail()  # 融资融券明细
    return {s.stock_code for s in securities if s.short_qty > 0}

# 空头信号过滤
def filter_shortable(top_100_codes: list, shortable: set, min_qty: int = 10000):
    """从超买100只中筛选可融券且余量充足的"""
    return [c for c in top_100_codes 
            if c in shortable and get_short_qty(c) >= min_qty]
```

### 阶段 3：优化因子 + 全面多空（6-12 个月后）

**目标 Sharpe**: 1.5+（稳定）

改进方向：
1. 增加因子数量（目前只用 MA60 偏离，可加 RSI、动量反转、资金流）
2. 机器学习因子合成（XGBoost/LightGBM 组合多因子）
3. 行业约束对冲（确保行业暴露接近0）
4. 动态调整空头比例（牛市减空，熊市加空）

## 风险点 & 对策

| 风险 | 描述 | 对策 |
|------|------|------|
| **融券被召回** | 券商强制平仓空头 | 保持20%备用空头池，随时替换 |
| **极端行情** | 空头持仓大涨（逼空） | 单只空头 ≤ 1.5%，触发1.5x涨幅止损 |
| **2024 牛市敞口** | 做空超买股在牛市中损失 | 牛市信号时降低空头比例至 50% |
| **流动性风险** | 空头无法平仓 | 只做空日均成交 >2亿的大盘股 |
| **期货基差** | 期货贴水/升水波动 | 以 IF（沪深300）为主，基差最小 |
