# 数据获取模块 · 架构设计总览

> 数据获取模块是整个量化系统的"感官"，负责从外部世界采集原始数据，
> 经过标准化后交给存储层。设计目标：**稳定、可扩展、可观测**。

---

## 模块定位

```
外部数据源                     数据获取模块                    存储层
─────────                     ──────────                     ─────
Tushare Pro  ──┐                                          ┌─→ ClickHouse
AKShare      ──┤   ┌────────────────────────────────┐     │
BaoStock     ──┼──→│  Fetcher → Cleaner → Writer    │─────┤─→ PostgreSQL
miniQMT      ──┤   │  (采集)   (清洗)    (写入)     │     │
Wind (预留)  ──┘   └────────────────────────────────┘     └─→ Redis
```

---

## 设计原则

| 原则 | 说明 |
|------|------|
| **数据源无关** | 上层只面向统一接口，不感知具体数据源 |
| **防御性编程** | 外部 API 随时可能超时/限频/返回脏数据，必须容错 |
| **幂等更新** | 同一天重复拉取不会产生重复数据 |
| **可观测** | 每次采集有日志、耗时、数据量记录，异常有告警 |
| **渐进式构建** | Phase 0 先跑通 Tushare 日K线，后续逐步加数据源 |

## 模块全景

```
data/
├── fetchers/                  # 数据采集层
│   ├── base.py                # 抽象基类 + 通用装饰器
│   ├── tushare_fetcher.py     # Tushare Pro 实现
│   ├── akshare_fetcher.py     # AKShare 实现
│   ├── baostock_fetcher.py    # BaoStock 实现
│   └── qmt_fetcher.py         # miniQMT 实时行情（预留）
│
├── cleaners/                  # 数据清洗层
│   ├── base.py                # 清洗器基类
│   ├── price_cleaner.py       # 行情数据清洗（复权/停牌/涨跌停标记）
│   ├── fundamental_cleaner.py # 基本面数据清洗（财报时间对齐）
│   └── reference_cleaner.py   # 基础信息清洗（退市/ST 标记）
│
├── writers/                   # 数据写入层
│   ├── base.py                # 写入器基类
│   ├── clickhouse_writer.py   # ClickHouse 写入（K线/因子）
│   ├── postgres_writer.py     # PostgreSQL 写入（基本面/元数据）
│   └── redis_writer.py        # Redis 写入（实时快照）
│
├── pipeline/                  # 流水线编排
│   ├── daily_pipeline.py      # 每日收盘后自动更新
│   ├── backfill_pipeline.py   # 历史数据回填
│   └── scheduler.py           # 定时调度器
│
├── quality/                   # 数据质量
│   ├── checker.py             # 数据质量检查引擎
│   └── rules.py               # 检查规则定义
│
├── common/                    # 公共工具
│   ├── calendar.py            # A 股交易日历
│   ├── rate_limiter.py        # 速率限制器
│   ├── retry.py               # 重试策略
│   ├── models.py              # 数据模型定义（dataclass）
│   └── config.py              # 配置管理
│
└── config/                    # 配置文件
    ├── settings.yaml           # 主配置
    ├── sources.yaml            # 数据源配置
    └── schedules.yaml          # 调度配置
```

## 实现优先级

| 优先级 | 组件 | 说明 |
|--------|------|------|
| **P0** | TushareFetcher + PriceCleaner + ClickHouseWriter | 最小可用：日K线入库 |
| **P0** | 交易日历、配置管理 | 基础设施 |
| **P1** | 基本面数据链路（财务指标/估值） | 多因子策略需要 |
| **P1** | 每日定时更新 Pipeline | 自动化运维 |
| **P1** | 数据质量检查 | 数据可信度保障 |
| **P2** | AKShare/BaoStock 补充数据源 | 冗余 + 补充 |
| **P2** | 历史回填 Pipeline | 2010~至今全量回填 |
| **P3** | miniQMT 实时行情 | 实盘阶段才需要 |
| **P3** | Redis 实时写入 | 实盘阶段才需要 |
